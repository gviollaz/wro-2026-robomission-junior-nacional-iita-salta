# Contributing (IITA)

## Regla 1: todo va por Issues + PRs
- Crear Issue (bug/feature/test/research/hardware/docs)
- Crear rama
- Abrir PR y linkear Issue

## Regla 2: declarar IA y fuentes
Si se uso IA o fuentes externas, declararlo en el PR y agregar links.

## Regla 3: no subir secretos
Nunca commitear credenciales, tokens, passwords. Usar `.env` y GitHub Secrets.

## Regla 4: visibilidad private-first
Este repo es **privado** durante construccion.

## PR checklist minimo
- Compila / corre en Pybricks
- Evidencia (logs/fotos/videos)
- Docs actualizadas
- Review pedido a responsables (CODEOWNERS)
